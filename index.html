GeoJson Report

The geojson file is a JSON file. It has the same structure. There are key-value pairs where the keys are denoted as string with "", a delimeter of : is used to indicate the value is starting. The value has many forms, in the geojson can be found strings using "", lists using [], numbers as numbers, and more key-value pairs using {}. Respectively, in the file on the right examples can be seen with "type": "Feature", "coordinates": [...], [-79.40011464833961, 43.65795077691766], and "properties": {}.

Of course, the data represents physical things in a geojson. The nested key-value pairs of "geometry": {"coordinates": [...]} is the polygon drawn, with each list of two values representing xy coordinates of the point. The "type": "Polygon" key-value pair tells us the geojson represents a polygon.

In terms of size, the shapefile is larger in terms of storage space used. However, my comparison is with rather small files of around 1KB, so it is possible the metadata is a majority of the shapefile size. This means its possible for large datasets that shapefiles are more compact. The geojson is in plain text format, seemingly in plain ASCII encoding. The only file which is human-readable format is the .prj file, with the other three being in binary format.

Some advanatages of using a geojson are it is a plain text file, meaning its readable by humans, it uses JSON format, which is a standard format meaning it will compatible with alot of existing infrastructure. Another advantage is ease of editbility. Some disadvantages of a geojson could be is it similar to raster data, where each bit of data must be explcitilly defined, meaning its size will increase relative to vector based formats.
